# Sudoku.com Candidate Helper

这个工作区里放了一个给 `https://sudoku.com/zh/evil/` 用的候选数助手脚本：

- `sudoku-helper.user.js`

用法：

1. 打开 Sudoku 页面，等棋盘加载完成。
2. 直接把 `sudoku-helper.user.js` 的全部内容粘到浏览器控制台执行，或装到 Tampermonkey 作为 Userscript。
3. 页面右下角会出现“数独候选助手”面板，棋盘上会叠加候选数。

自动读取逻辑：

- 新版脚本会在 `document-start` 监听 sudoku.com 的题目接口，页面刚加载出棋盘时就能抓到题面。
- 如果你已经填过数字，会优先读取页面运行状态或存档里的当前盘面。
- 不需要先随便填一个数字来触发 localStorage。

Tampermonkey 自动更新地址：

- 安装/下载：`https://raw.githubusercontent.com/SeptYagu/sudoku-helper/main/sudoku-helper.user.js?raw=1`
- 脚本头里已经配置了 `@updateURL` 和 `@downloadURL`。

提示规则：

- 小号蓝色数字：这个空格当前合法的候选数。
- 绿色大号数字：排除法后只剩一个候选数字，可以确定填写。
- 黄色大号数字：这个数字在当前行、列或宫里只有这一个位置可放，可以确定填写。
- 红色边框或感叹号：当前盘面有冲突，或某空格没有合法候选数。
- 面板会显示推理思路，包括宫内指向、行列锁定、裸对/裸三/裸四、隐藏对/隐藏三、X-Wing 等候选删减步骤。

自动填写：

- 点“自动填写”会尝试把当前所有绿色/黄色确定数字填进网页棋盘。
- 自动填写会使用经过逻辑推理删候选后得到的确定数字，但不会试填或猜测；每填一格都会重新读盘并确认，失败会暂停。
- 普通蓝色候选数不会自动填写；如果盘面有冲突，脚本会停止自动填写。
- “自动填写速度”滑条可以控制每格填写间隔；脚本会在基础间隔上加入随机抖动，避免固定节奏过快。
- 自动填写运行中按钮会变成“停止填写”，再点一次会尽快中断。

如果页面以后改版导致自动读取失败：

1. 先点助手面板里的“刷新”。
2. 仍失败就点“诊断”，把控制台里的 `[SudokuCandidateHelper] diagnostics` 发回来。
3. 临时兜底可以打开“手动盘面”，粘贴 81 位盘面，`0` 或 `.` 表示空格。
